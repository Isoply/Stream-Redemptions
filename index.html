Stream Redemptions Documentation

This is where all the documentation for the stream redemptions application will be stated.

When opening the app, you have 6 options:

  • Wheel

      -Takes you to the Wheel section.

  • Random Loadout

      -Takes you to the Random Loadout section.

  • Marbles

      -Takes you to the Marbles section.

  • Apply Current Redeem

      -Applies Current Redeem applies every redeemed reward that has not been activated yet an has been saved (This will also happen if you go into any specified section).

  • Documentation

      -Opens this website.

  • Quit

      -Quits the application.

Every section has an invisible button in the top right to open the settings.
They also all have a button to swap to another app and a button to login to Twitch (that dissapears when logged in) and a button to open the folder of the specified section.

Wheel Documentation

When in the Wheel section, these are all your options:

Framerate
  -The framerate at which this section runs.
Reward Name
  -The reward name your viewers can redeem to add to the wheel (Case sensitive).
Wheel Start Speed
  -The speed at which the wheel start to spin.
Wheel Deceleration Speed
  -The speed at which the wheel decelerates when spinning.
Tick Volume
  -How loud the ticks sound when a wheel section passes.
Test Wheel Segments
  -How many segments the wheel has (for testing purposes). Does get rid of any existing segments!
Delete After Selection
  -Deletes segments when they get selected by the wheel if selected.
View Segments
  -Displays all the segments currently on your wheel. You can also add your own if you'd like.
View Color Options
  -Displays all the colors of each segment. Change to your hearts content.



You can start spinning the wheel by pressing the button on the wheel when you have segments on your wheel.
The images of the wheel can be changed by going into the folder and going to ".../Wheel/Images/" and in this folder add images called
"Button.png", "Pointer.png" and "Wheel.png"

You can also find all your saved setting in the ".../Wheel/" folder in "Save.save",the segments in "WheelSegments.save" and segment colors in "WheelColors.save".
If any of these do not exist, then that means you have not changed/added anything for it in the app.

Random Loadout Documentation

When in the Random Loadout section, these are all your options:

Framerate
  -The framerate at which this section runs.
Reward Name
  -The reward name your viewers can redeem to randomize your loadout. (Case sensitive).
Loadout Dropdown
  -This is a dropdown that displays all the text files that you have in your folder.
Get Random Loadout
  -Gives you a random loadout of the currently selected Loadout file.
No Dupes
  -If selected, makes sure no item shows up twice.
Enable Queue
  -If enables, the first redeem will randomize your loadout, any future ones will pile up in a queue on the bottom right and can be clicked to use 1.
Clear Queue
  -Clears the current queue.



On the bottom right, where the queue also appears, there is always an invisible button to randomize your loadout.
All your loadouts get taken out of the folders, which are in ".../Loadout/Loadouts/".

When you open loadouts for the first time, it will automatically make an example file for class based loadouts and regular loadouts.
Just name the file you're making to whatever you'd like, be it game or something else.
Any file for loadouts can follow this formatting:

[Number of items to be pulled from this file]
item 1
item 2
item 3
~ (This is an indicator of a new section, you can also add a number behind it if you want to take a different number from this secction than stated at the top.)
item 4
item 5
item 6


Anything incased in [] is something that should be replaced with what is stated inside.

For anything class based, you want to create a folder named after the game or something else.
Then inside you want to create text files for each class/individual loadout following the same previous formatting.


You can also change how the displayed loadout backgrounds look by changing the "Background.png" in ".../Loadout/Images/".
Or you can make a folder in ".../Loadout/Images/" with the game/something else name and put a "Background.png" in there for a specific game.
Also, in that same folder, you can add images with the same name as items to load the image beside the item like shown in a previous screenshot.
For any claass based folders like explained previously, you will also need to make class specific folders inside the game/something else folder.
Any save data is saved in the original loadout folder under ".../Loadout/Save.save".

Marbles Documentation

When in the Marbles section, these are all your options:

Framerate
  -The framerate at which this section runs.
Reward Name
  -The reward name your viewers can redeem to add to the wheel (Case sensitive).
Change Structure
  -Changes to any other structures currently available.
Win Chance (Pin Based Structure)
  -The chance for a bal lto land in the winning zone.
Allow Multiple Entries
  -If a user iss allowed to enter multiple times.
Marble Size
  -How large a marble is when spawned in.
Row Count (Pin Based Structure)
  -How many rows of pins the structure has.
Pin Count (Pin Based Structure)
  -How many pins are in a row.
Pin Size (Pin Based Structure)
  -How large each pin is.
Spawn Marble
  -Open/Close Door (Non-Pin Based Structure)
Opens/Closes the door to let the marbles fall into the structure.
  -Spawns a marble.
Spawn Marbles
  -Spawns a specified amount of marbles.
Clear Marbles
  -Clears all marbles.



To add any specific images/colors to the marbles, go to ".../Marbles/Images/Marbles/" and any .png file entered here will be randomly applied to the marbles.


Any save data is saved in ".../Marbles/" as "Save.save".
